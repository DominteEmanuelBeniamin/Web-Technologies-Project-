1. Introduction

1.1 Purpose

The purpose of this project is to develop a web-based tool that enables users to provide feedback for various entities (such as events, people, geographic locations, products, services, artistic artifacts, etc.) in an anonymous manner. Feedback, in this context, is understood as expressing a specific emotion according to the Plutchik model. The application, through a REST/GraphQL API, should manage the entities to be evaluated and the specific responses for each (editable by the initiator of a feedback collection request). After the completion of the period during which a form is accessible to individuals for reviewing a particular entity, statistics of interest regarding each category of evaluated entities according to each recorded emotion will be presented. Multiple criteria will be considered, such as user group, evaluation time period, subcategories of entities, positive/negative characteristics based on the shared emotion, etc. The generated reports will be available in HTML, CSV, and JSON formats. Additionally, implementing attractive data visualization methods as described in the provided bonus resource will be pursued.

1.2 Product Scope

The product scope includes the following features and functionalities:

  • Feedback Collection: Users can submit feedback for various entities anonymously. They will have the option to specify their emotions according to the Plutchik model.
  • Entity Management: The application should allow administrators to manage the entities that are open for feedback. This includes adding, editing, and removing entities from the system.
  • Feedback Response Management: Initiators of feedback collection requests should have the ability to manage the responses received for each entity. They can view, edit, and analyze the feedback data.
  • Statistics Generation: After the feedback collection period for an entity concludes, the system will generate statistics regarding the emotions expressed for that entity. These statistics will be categorized based on various criteria such as user demographics, time periods, and subcategories of entities.
  • Report Generation: Reports summarizing the feedback statistics will be generated in HTML, CSV.
