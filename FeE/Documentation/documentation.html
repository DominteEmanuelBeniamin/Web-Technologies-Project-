1. Introduction

1.1 Objectives

The primary objective of this project is to develop a user-friendly web-based platform that facilitates feedback across diverse domains, including events, individuals, locations, products, services, and artworks. Feedback submissions will be anonymized and categorized based on various emotions using the Plutchik model, which offers a comprehensive framework for understanding and organizing human emotions.

Utilizing HTML, CSS, PHP, and other pertinent technologies, we aim to create an intuitive and aesthetically pleasing interface for users to engage with the platform seamlessly. The integration of these technologies will ensure efficient data presentation and management, enhancing the overall user experience.

Powered by a REST/GraphQL API, the platform will manage the collection and organization of feedback from users. Post-collection, comprehensive reports will be generated, leveraging insights derived from the Plutchik model. These reports will provide statistical analyses, considering factors such as user demographics, timing of feedback submission, and sentiment analysis based on expressed emotions.

The generated reports will be available in multiple formats, including HTML, CSV, and JSON, catering to diverse user preferences. Additionally, efforts will be made to present the data in visually engaging formats, optimizing user comprehension and engagement.

By incorporating the principles of the Plutchik model and utilizing HTML, CSS, PHP, and related technologies, this project aims to deliver a robust feedback platform that not only facilitates meaningful interactions but also ensures accessibility and usability for all users.

1.2 Product Scope

Our product encompasses a comprehensive suite of features and functionalities tailored to facilitate seamless feedback management and analysis. Key components of our product include:

  • Feedback Collection: Users can effortlessly provide feedback for a diverse range of entities, ensuring anonymity and ease of expression. Leveraging the Plutchik model, users can articulate their emotions effectively, enriching the feedback process.
  • Entity Management: Administrators are equipped with robust tools to efficiently manage entities available for feedback. This entails seamless additions, modifications, and removals from the system, ensuring flexibility and relevance.
  • Feedback Response Management: Initiators of feedback collection exercise full control over received responses for each entity. This includes the ability to review, edit, and analyze feedback data, empowering stakeholders to derive actionable insights.
  • Statistics Generation: At the culmination of the feedback collection period, our system generates comprehensive statistics delineating expressed emotions for each entity. These statistics are intelligently categorized based on various criteria such as user demographics, evaluation timeframes, and entity subcategories, furnishing stakeholders with invaluable insights.
  • Report Generation: Automated generation of comprehensive reports summarizing feedback statistics in user-friendly formats such as HTML and CSV. These reports facilitate easy interpretation and dissemination of actionable insights, fostering informed decision-making.

By encapsulating these core functionalities, our product aims to streamline the feedback management process, enhance stakeholder engagement, and empower informed decision-making across diverse domains.

3. External Interface Requirements

3.1 User Interfaces

The user interface (UI) for the system is crafted to provide an optimal user experience through intuitive layout and efficient navigation. The design emphasizes clarity and simplicity, enabling users to interact with the system effortlessly.

Sign In/Sign Out Features

The Sign In and Sign Out functionalities are critical for maintaining user session security and providing appropriate access permissions across the system. Users are required to sign in to access personalized features and administrative functions such as adding products.

  • Sign In:

    Users are prompted to sign in through a secure, streamlined login page that offers options for password-based or social media logins. Successful authentication grants access to the full spectrum of system features, based on user roles.

  • Sign Out:

    When users sign out, the system immediately terminates the session and clears any temporary data, ensuring user information remains confidential and preventing unauthorized access.

Home Page

The Home Page is the gateway to the system, providing direct links to all main features including product views, archival access, and new product additions. It serves as the central hub for navigation.

  • Navigation Bar:

    Features straightforward links to Home, View Products, Add Product, View Archive, and Documentation, facilitating quick access from any part of the system.

  • Main Section:

    Highlights current promotions or important updates through a prominent display, ensuring users are immediately informed of key happenings and offers.

View Products Page

This page allows users to explore and review a comprehensive list of products, detailed with high-quality images and essential information, enhancing the shopping experience.

  • Product List:

    Each product is displayed with an image, name, and brief description in a clean, organized layout, which facilitates easy browsing and selection.

Add Product Page (Requires Sign In)

Accessible only to authenticated users, this page allows the addition of new products. It is specifically designed to capture all necessary information through a methodical form, ensuring that all listings are comprehensive and useful.

  • Product Submission Form:

    Incorporates fields for uploading product images, and detailed descriptions, and allows sellers to set categories and prices, which helps maintain a uniform product appearance on the platform.

  • Feedback Mechanism:

    Sellers can add customizable questions to gather targeted feedback from customers, enhancing product quality based on user responses.

View Archive Page

The View Archive page is dedicated to providing access to products that are no longer available for purchase but can still be referenced for information or feedback.

  • Archived Products Display:

    Archived items are listed in a manner similar to active products but are clearly marked to distinguish them from currently available items.

Documentation Page

This page is a repository of resources about the platform, offering detailed guides and FAQs to help users understand and efficiently use the system.

  • Searchable Content:

    Includes a search feature enabling users to directly find specific topics or instructions based on keywords, which enhances the utility of the documentation.

Responsive Design

The UI adjusts effectively across different devices and resolutions, ensuring a consistent and professional user experience whether accessed via desktop, tablet, or smartphone. This responsive design approach guarantees that all features are usable and visually appealing on any device.
